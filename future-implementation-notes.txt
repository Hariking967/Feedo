Open Items
==========

[FI-001] Cross-role lifecycle event ledger (Supplier <-> NGO/Receiver <-> Volunteer)
Status: Implemented (needs staged validation)
Why deferred:
- End-to-end actor journey cannot be fully validated right now in a shared live environment.
Current gap:
- We have listing/emergency/bulk/proof flows, but no single event ledger for every role action on the same listing/event timeline.
Future implementation:
- Add table: listing_lifecycle_event
- Write events from all role APIs (supplier create/update, volunteer accept/pickup/dropoff, receiver confirm, NGO intake/close, system auto-expire).
- Add read endpoint for supplier timeline and role-filtered activity.
Planned input contract:
- eventId: string
- listingId: string | null
- donationEventId: string | null
- supplierUserId: string
- actorUserId: string
- actorRole: supplier|volunteer|receiver|ngo|recipient|system
- eventType: listing_created|emergency_triggered|volunteer_assigned|picked_up|delivered|confirmed|expired|cancelled|proof_uploaded|allocation_split|allocation_completed
- statusAfter: string | null
- payload: json
- occurredAt: timestamp
Planned output contract:
- timeline[] sorted desc with actor, role, eventType, statusAfter, payload, occurredAt

[FI-002] Real-time live event propagation for cross-role state changes
Status: Pending
Why deferred:
- No reliable multi-client real-time session test was run for all roles.
Current gap:
- Role actions update DB state but there is no guaranteed shared subscription channel for supplier/ngo/volunteer/receiver views.
Future implementation:
- Add Supabase Realtime channel per listing/event (for example: listing:<id>, event:<id>).
- Publish lifecycle events and status updates to channels.
- Add client subscribe/unsubscribe in role dashboards.
Planned input contract:
- channelKey: string
- eventName: string
- payload: { listingId?, donationEventId?, actorRole, eventType, statusAfter?, metrics? }
Planned output contract:
- role dashboards receive same event packet with acknowledged update timestamp.

Completed Later
===============
- [Done] Cross-role lifecycle event ledger (FI-001)
	- Added shared lifecycle writer utility `src/lib/lifecycle-events.ts`.
	- Added timeline read API `GET /api/lifecycle` with listing/event/actorRole filters.
	- Added lifecycle writes in supplier listing, emergency trigger, bulk allocation, and volunteer workflow routes.
	- Event table target: `listing_lifecycle_event` (Supabase-backed, graceful no-op fallback if missing).
- [Done] Volunteer emergency queue + action workflow (FI-006)
	- Added `GET /api/volunteer/emergency-queue` for unassigned/assigned/expiring emergency tasks.
	- Added `POST /api/volunteer/emergency-action` for accept/reject/en-route/picked-up/delivered/unable transitions.
	- Each action updates listing state and emits lifecycle events for audit traceability.
- [Done] Proactive receiver-need to supplier-prompt loop
	- Added `POST/GET /api/receiver/needs` for advance need posting and history.
	- Added `GET /api/supplier/need-prompts` for supplier-side targeted prompt visibility.
	- Added Supabase tables `receiver_need_request` and `supplier_need_prompt`.
	- Added recipient dashboard need-posting panel and supplier/donor prompt panel for visible frontend behavior.
- [Done] Crisis-aware priority receiving
	- Added crisis signal detection in `GET /api/receiver/feed` using OpenWeather + optional manual `crisis_zone` checks.
	- Added survival-first ranking mode during crisis emphasizing fastest safe route and highest urgency.
	- Added consumer UI crisis banner and urgent redistribution state in active `src/app/page.tsx` flow.
	- Extended `POST /api/receiver/needs` to broaden supplier targeting and dispatch crisis nearby alerts via Firebase/outbox.
- [Done] Live inbound tracking with suitability warnings
	- Added `GET /api/receiver/inbound-tracking` for stage, ETA, assigned volunteer, route delay and suitability warning snapshots.
	- Added major stage change push dispatch (`pickup_confirmed`, `en_route`, `nearing_arrival`, `delayed`) via Firebase/outbox fallback.
	- Added consumer tracking UI block showing inbound stage, pickup status, volunteer assignment, and spoilage-risk warnings.
